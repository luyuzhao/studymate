## 一、项目概述

### 1.1 选题背景

大学生在学习过程中需要同时管理课程、任务、复习、习惯等多方面事务，现有工具（如日历、备忘录）功能分散，缺乏学习场景的深度整合。StudyMate Pro 旨在打造一款一站式学习效率工具，将课程管理、智能任务、番茄专注、闪卡记忆等功能有机整合，帮助学生提升学习效率。

### 1.2 项目目标

- 开发一个可运行、有实际用户价值的跨平台移动应用
- 支持 Android 5.0+（API 21）手机和平板设备
- 遵循 Material Design 3 设计规范，支持深色模式
- 全程使用 AI 辅助编程工具，标注 AI 生成与人工修改部分

### 1.3 功能概述

本应用包含 14 个功能模块：

1. Dashboard 数据概览首页
2. 课程管理（含 GPA 计算、上课时间设置）
3. 任务看板（三栏视图、子任务、重复任务）
4. 番茄专注计时器（多种时长、统计图表）
5. 闪卡记忆（SM-2 间隔重复算法）
6. Markdown 笔记（课程关联、全文搜索）
7. 习惯打卡（热力图、连续天数）
8. 智能记账（分类饼图统计）
9. 日历课表（月/周视图、课程+待办聚合）
10. 学习报告（本周/本月汇总、图表可视化）
11. 成就勋章系统（19 项成就、自动检测）
12. OCR 拍照导入（中英文识别、转闪卡/笔记）
13. 个人中心（注册登录、备份恢复、主题切换）
14. 数据安全（密码加盐哈希、自动迁移）

## 二、技术方案

### 2.1 技术选型

| 技术 | 版本 | 用途 |
|------|------|------|
| Flutter | 3.x | 跨平台 UI 框架 |
| Dart | 3.5+ | 编程语言 |
| Riverpod | 2.x | 状态管理 |
| Hive | 2.x | 本地 NoSQL 数据库（业务数据） |
| SQLite (sqflite) | — | 关系数据库（用户账户） |
| fl_chart | — | 图表可视化 |
| table_calendar | 3.x | 日历组件 |
| google_mlkit_text_recognition | 0.13.x | OCR 文字识别 |
| flutter_animate | 4.x | 声明式动画 |

**选型理由**：
- Flutter 实现一套代码多平台运行，开发效率高
- Riverpod 相比 Provider 更灵活、可测试性更好
- Hive 无需 SQL 定义即可存储复杂对象，适合快速原型开发
- SQLite 用于用户账户确保数据可靠性

### 2.2 架构设计

```
┌─────────────────────────────────────┐
│              UI Layer               │
│  (Pages / Widgets / Animations)     │
├─────────────────────────────────────┤
│          State Management           │
│  (Riverpod StateNotifierProvider)   │
├─────────────────────────────────────┤
│            Data Layer               │
│  ┌──────────┐  ┌─────────────────┐  │
│  │   Hive   │  │    SQLite       │  │
│  │(业务数据) │  │ (用户账户)      │  │
│  └──────────┘  └─────────────────┘  │
└─────────────────────────────────────┘
```

### 2.3 核心算法

**SM-2 间隔重复算法**（闪卡模块）：

```
输入: 用户评分 rating (0=Again, 2=Hard, 3=Good, 5=Easy)
处理:
  if rating == 0:  重置为学习中，间隔 = 0
  if rating == 2:  间隔 × 1.2，EF 不变
  if rating == 3:  间隔 × EF，EF 微调
  if rating == 5:  间隔 × EF × 1.3，EF 增加
输出: 更新 nextReviewDate = now + intervalDays
```

## 四、功能实现详情

### 4.1 数据持久化

使用 Hive 作为主要本地数据库，为每个数据模型注册 TypeAdapter：
- 9 种数据模型：Course, Task, Flashcard, Note, Habit, Expense, PomodoroRecord, Achievement, UserProfile
- 9 个 Hive Box 并行打开，启动速度优化
- 用户账户使用 SQLite 存储，确保可靠性
- 支持 Hive → SQLite 自动数据迁移

### 4.2 状态管理

采用 Riverpod StateNotifierProvider 模式：
- 每个功能模块一个 Provider
- Provider 间通过 `ref.watch` 实现数据联动
- 用户切换时自动重新加载对应数据

### 4.3 UI 设计

- Material Design 3 完整适配
- 支持亮色/深色/跟随系统三种主题
- 使用 flutter_animate 实现入场动画
- 各模块专属品牌色区分
